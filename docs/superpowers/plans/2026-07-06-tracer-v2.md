## PHASE 1 — Fingerprint tab (build now)

Cross-group operator correlator. Pure DB read, zero Horizon calls. Scores every PAIR of asset groups 0–100 for "likely same operator", with evidence rows.

### Scoring model (locked)

Aggregation = dampened probabilistic-OR over deduped evidence items:
`score = round(100 × (1 − Π(1 − wᵢ)))`.

Constants (all in one block at top of `fingerprint.ts`):

```typescript
const ROLE_WEIGHT: Record<GroupMemberRole, number> = {
  issuer: 0.6, distributor: 0.6,
  intermediary: 0.5, bank: 0.5, creator: 0.5, withdrawal: 0.45,
  destination: 0.3, service: 0.15, other: 0.15,
};
const W_TOP_DESTINATION = 0.4;  // signal 2
const W_HOME_DOMAIN     = 0.35; // signal 3
const W_LINEAGE         = 0.55; // signal 4
const SINGLE_ITEM_CAP   = 0.6;  // one item alone can never exceed 60
const DOMAIN_CLUTTER_K  = 8;    // domains in >8 groups: skip evidence row entirely
const MIN_SCORE_DEFAULT = 25;   // UI hides below this

// IDF dampening: k = number of groups (or distinct assets for signal 2) containing the entity
const damp = (k: number) => 1 / (1 + Math.log2(Math.max(k, 2) / 2));
// k=2→1.0, k=4→0.5, k=8→0.33, k=16→0.25
```

Rules:
- **Short-circuit:** if an address is role `issuer` OR `distributor` in BOTH groups of the pair → pair score = 100, tier `"confirmed"`. (Same account issues/distributes both assets = same operator.)
- **Signal 1** shared address: item weight = `max(ROLE_WEIGHT[roleInA], ROLE_WEIGHT[roleInB])` (an address may hold different roles in each group).
- **Signal 2** shared top-destination: address present in `topDestinations` of saved analyses for ≥2 different assets → weight `W_TOP_DESTINATION`. `k` = distinct assets containing it.
- **Signal 3** shared home domain: same non-empty `homeDomain` on members of both groups → weight `W_HOME_DOMAIN`. If domain appears in > `DOMAIN_CLUTTER_K` groups, do not emit.
- **Signal 4** shared lineage: a known creator whose `CreatorChild.childAddress` set intersects members of BOTH groups → weight `W_LINEAGE`.
- **Dedup by entity:** key each evidence item by the shared entity (address or domain). If one address fires multiple signals, keep only its single highest effective weight — no double counting.
- Effective weight per item = `min(baseWeight × damp(k), SINGLE_ITEM_CAP)`.
- **Tiers:** `100`=confirmed, `80–99`=strong, `50–79`=moderate, `25–49`=weak, `<25`=hidden by default.
- **Scope:** only pair groups on the SAME `network`.

```ts
export interface BulkTraceRow {
  address: string;
  status: "pending" | "done" | "error" | "not-found";
  result?: TraceResult;   // from @/lib/intermediary-tracer/types
  error?: string;
}
export interface BulkTraceOptions {
  addresses: string[];
  horizonUrl: string;
  windowSec: number;
  tolerancePct: number;
  knownIntermediaries: Map<string, string>;
  signal: AbortSignal;
  concurrency?: number;                       // default 4
  onLog?: (msg: string) => void;
  onResult: (row: BulkTraceRow) => void;      // called on pending AND on settle (merge by address)
  // injectable for tests — defaults to the real tracer:
  traceFn?: typeof traceAccountOrigin;
}
export async function runBulkTrace(opts: BulkTraceOptions): Promise<BulkTraceRow[]>;
```

Behavior:
- Dedup addresses (preserve order). Emit each as `{status:"pending"}` via `onResult` up front.
- Worker pool, `concurrency` workers (default 4), same shape as `scanIntermediaryCreations`:
  ```ts
  const queue = [...addresses];
  async function worker() {
    while (queue.length && !signal.aborted) {
      const addr = queue.shift()!;
      // call traceFn, build row, onResult(row)
    }
  }
  await Promise.all(Array.from({length: Math.min(concurrency, addresses.length)}, worker));
  ```
- Per address: `traceFn(horizonUrl, addr, windowSec, tolerancePct, knownIntermediaries, signal, onLog)`. `null` → `status:"not-found"`. Throw → `status:"error"`, `error: message` (skip if `signal.aborted`). Success → `status:"done"`, `result`.
- Emit settled row via `onResult`. Return the final `BulkTraceRow[]` (order = input order).
- Never throw on individual failures; only the abort short-circuits the loop.

```ts
export type GraphNodeKind = "group-member" | "intermediary" | "creator" | "creator-child" | "destination";
export interface GraphNode {
  id: string;              // address (dedup key)
  kinds: GraphNodeKind[];  // multi-badge — a node can be several things
  roles: string[];         // group roles held (issuer/distributor/bank/...)

  network: string;
}
export interface GraphEdge {
  source: string;          // address
  target: string;          // address
  kind: "creator-child" | "intermediary-created" | "distrib-destination" | "co-membership";
  weight: number;          // XLM for distrib-destination, else 1
}
export interface GraphData { nodes: GraphNode[]; edges: GraphEdge[]; }
export interface GraphFilters {
  network?: string;                 // restrict to one network
  groupId?: string;                 // only nodes/edges touching this group
  kinds?: GraphNodeKind[];          // include only these node kinds
  minEdgeWeight?: number;           // drop lighter edges
  focusAddress?: string;            // N-hop neighborhood only
  focusHops?: number;               // default 2
}
export interface GraphInput {
  groups: AssetGroup[];
  knownIntermediaries: { address: string; name: string }[];
  knownCreators: { address: string; name: string }[];
  creatorChildren: CreatorChild[];
  analyses: SavedAnalysis[];
  filters?: GraphFilters;
}
export function buildGraph(input: GraphInput): GraphData;
```

Behavior:
- **Nodes**, deduped by address (a node accumulates multiple `kinds`/`roles`):
  - group members → kind `group-member`, push their role into `roles`; `label` from member label; `network` from group.
  - known intermediaries → kind `intermediary`; label = name.
  - known creators → kind `creator`; label = name.
  - creator children → kind `creator-child`.
  - saved-analysis `topDestinations` addresses → kind `destination`.
- **Edges:**
  - `creator-child`: `CreatorChild.creatorAddress` → `childAddress`, weight 1.
  - `intermediary-created`: `CreatorChild.viaIntermediary` (if set) → `childAddress`, weight 1.
  - `distrib-destination`: for each analysis, each `distribAddresses[]` → each `topDestinations[].address`, weight = `totalXlm`.
  - `co-membership`: for each group, connect members pairwise? NO — too dense. Instead connect each non-issuer member → the group's issuer (star), weight 1, kind `co-membership`. (Skip if group has no issuer.)
- **Filters** applied after building: network match; `groupId` keeps only nodes that are members of that group + edges between kept nodes; `kinds` filter; `minEdgeWeight` drops edges (then prune orphan nodes); `focusAddress` → BFS `focusHops` (default 2) from that address, keep only reached nodes + their edges.
- Only include an edge if BOTH endpoints survive node filtering.
